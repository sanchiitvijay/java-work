Functional Requirements Specification for Library Management System
(Internal – Facing)
Module: Admin + Staff (Librarian + Manager)
0. Current System Overview
The following functionality is already available in the base build provided to associates:
Backend (Spring Boot)
Services:
Eureka Discovery Server (service registry)
API Gateway (routes requests to microservices)
Catalog Service (currently active)
Catalog Service Entities (with JPA + H2):
Book → id, title, author, isbn, category, status
BookCopy → id, bookId, rackNo, status
Member → id, name, email, phone, membershipStatus
Staff → id, name, role, email, status
Controllers (working):
BookController (/api/admin/books)
BookCopyController (/api/admin/books/{bookId}/copies)
MemberController (/api/staff/members)
StaffController (/api/admin/staff)
Filters / Queries:
Search books by title, author, category
Filter book copies by status
Repositories:
BookRepository
BookCopyRepository
MemberRepository
StaffRepository
1. Overview
The Admin + Staff module provides internal-facing functionality for library operations.
Librarians manage book issue/return and member interactions.
Managers monitor library usage and reports.
Admins manage catalog and staff accounts.
2. Scope
Book catalog and inventory management
Member registration and membership control
Issue, return, and fine handling
Reporting and monitoring
3. Actors
Actor
Role in System
Admin
Manages books, copies, staff accounts
Librarian
Issues/returns books, manages members
Manager
Reviews reports and library performance

5. Non-Functional Requirements
NFR-01: Role-based access control
NFR-02: Concurrent multi-user support

6. Use Cases
UC-01: Librarian Issues Book
Precondition:
Member is active
Book copy is available
Steps:
Librarian selects member
Searches for book
Issues available copy
System updates copy status to ISSUED
UC-02: Manager Generates Monthly Report
Precondition:
Library has transaction data
Steps:
Manager opens Reports module
Selects date range
Selects report type
System displays charts and tables
